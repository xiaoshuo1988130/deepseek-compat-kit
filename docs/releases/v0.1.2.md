# DeepSeek CompatKit v0.1.2

Small feature release for deterministic schema repair, endpoint capability checks, and print-only adoption recipes.

## Install

```bash
npx deepseek-compat-kit@0.1.2 --help
```

## Added

- Added `compile-schema` to convert generated Zod/Pydantic/JSON Schema output into a DeepSeek strict-mode compatible shape.
- Added compile reports with:
  - removed unsupported constraints
  - added `required` fields
  - enforced `additionalProperties: false`
  - `system_prompt_appendix`
  - `post_validation_plan`
- Added `probe` to generate a small functional capability report for official, relay, or self-hosted OpenAI-compatible endpoints.
- Added print-only `recipes opencode` and `doctor --target opencode --print`.
- Added OpenCode + DeepSeek recipe docs.

## Boundaries

- `probe` is a functional compatibility check, not a benchmark, load test, or model quality evaluation.
- `doctor` is intentionally print-only in this release. It does not scan or modify OpenCode configuration files.
- The OpenCode recipe is `DOCS_ONLY`; live end-to-end validation is pending.

## Tests

- Added regression coverage for `compile-schema`.
- Added mock-upstream coverage for `probe`.
- Added CLI coverage for `recipes` and `doctor`.
- CI/local verification should run:

```bash
npm test
npm run lint
npm run test:examples
```
