# DeepSeek CompatKit v0.1.4

Reliability release for local publishing, endpoint probes, provider matrices, and conservative proxy diagnostics.

## Install

```bash
npx deepseek-compat-kit@0.1.4 --help
```

## Added

- Added real-endpoint validation guidance and an example provider matrix report.
- Added an OpenRouter relay recipe and adoption-doctor support.
- Added probe header support with redacted reporting:
  - `--header "Name: Value"`
  - `--header-env "Name=ENV_VAR"`
- Added probe baseline comparison, warning gates, selected checks, check presets, timeout controls, and richer terminal summaries.
- Added matrix directory input, regression gates, required capability gates, and display names.
- Added diagnose JSON/Markdown output, original JSONL line references, proxy diagnostic findings, and `--fail-on-warn`.
- Added proxy diagnostics JSONL logging, timeout handling, `/health`, state TTL controls, and conservative reasoning restoration checks.
- Added npm package smoke coverage so release packages include docs and runnable examples without vendored `node_modules`.
- Added local release scripts:
  - `npm run verify`
  - `npm run pack:check`
  - `prepublishOnly`

## Changed

- `compile-schema` gained check mode, Markdown reports, and post-validation guidance for constraints moved out of strict JSON Schema.
- CLI option parsing now accepts `--flag=value` and handles positional inputs more predictably across commands.
- Inventory and doctor reports now include scan limits, target recommendations, and safer secret redaction.
- README docs now separate mock/local validation from live endpoint certification.

## Boundaries

- Doctor and recipes remain print-only. They do not edit third-party tool configuration or extension storage.
- Proxy reasoning restoration remains conservative. If matching is incomplete or ambiguous, it warns instead of guessing.
- Live framework integrations remain documented recipes unless backed by explicit endpoint or framework validation reports.

## Local Verification

```bash
npm run verify
npm run pack:check
```

The current package dry-run includes docs, fixtures, and runnable examples while excluding example `node_modules`.
