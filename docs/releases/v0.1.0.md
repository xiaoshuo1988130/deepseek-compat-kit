# DeepSeek CompatKit v0.1.0

First public alpha with a working local proxy.

## Install

```bash
npx deepseek-compat-kit@0.1.0 --help
```

## Highlights

- Added `proxy [--port 8787] [--upstream https://api.deepseek.com]`.
- Added stateful best-effort `reasoning_content` capture and injection for multi-turn tool calls.
- Added request-time strict schema diagnostics in terminal logs and response headers.
- Kept `diagnose`, `lint-schema`, `sanitize`, and `replay` available as CLI commands.
- Published npm package: `deepseek-compat-kit@0.1.0`.

## Boundary

The proxy is not a stateless magic fix. It can only replay `reasoning_content` when the relevant requests and responses passed through the proxy from the beginning of the conversation.

If a framework sends an already-broken `messages` array where `reasoning_content` was lost before DeepSeek CompatKit saw it, the proxy can diagnose the missing field but cannot reconstruct it from nothing.

## Quickstart

```bash
DEEPSEEK_API_KEY=sk-... npx deepseek-compat-kit@0.1.0 proxy --port 8787
```

Then set your OpenAI-compatible client base URL to:

```text
http://127.0.0.1:8787/v1
```
