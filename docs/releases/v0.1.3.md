# DeepSeek CompatKit v0.1.3

Adoption-doctor and capability-probe release focused on faster local setup triage.

## Install

```bash
npx deepseek-compat-kit@0.1.3 --help
```

## Added

- Added print-only adoption recipes for:
  - Cline
  - OpenAI JS SDK
  - LangChain JS
  - legacy Roo Code installations
- Added `doctor --target auto --path .` to generate one combined adoption report from locally detected targets.
- Added inventory target detection for OpenCode, Cline, Roo Code, OpenAI JS SDK, and LangChain JS.
- Added inventory recommendations that point to the next `doctor`, `probe`, or secret-handling step.
- Added richer `probe` capability reports with:
  - per-capability status
  - `profile_guidance`
  - check impact and recommendations
- Added `probe --profile official|openai|relay|self-hosted` with aliases such as `deepseek`, `gateway`, `vllm`, and `ollama`.
- Added endpoint diagnostics for `probe`, including normalization when a user passes a full `/chat/completions` URL instead of a base URL.
- Added dedicated docs for capability probe behavior and adoption doctor behavior.

## Boundaries

- Doctor and recipes remain print-only. They do not edit OpenCode, Cline, Roo Code, SDK, or framework configuration files.
- `doctor --target auto` requires an explicit local `--path` and only uses inventory heuristics.
- Roo Code support is documented as a legacy/installed-copy recipe because current official docs show a sunset notice.
- `probe` is still a functional compatibility check, not a benchmark, load test, or model quality evaluation.

## Tests

- Added regression coverage for:
  - Cline and Roo Code recipes.
  - OpenAI JS and LangChain JS recipes.
  - auto doctor report generation.
  - inventory target recommendations.
  - probe capability profile guidance.
  - endpoint normalization from `/chat/completions` to a base URL.
- CI/local verification should run:

```bash
npm run lint
npm test
npm run test:examples
npm run pack:check
```

Before publishing from a local machine, run:

```bash
npm run verify
```

`npm publish` also runs `prepublishOnly`, which executes local verification and a package dry-run.
